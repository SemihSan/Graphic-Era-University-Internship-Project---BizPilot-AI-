# Data / Veri

## English

This directory contains project data for BizPilot AI.

### Company Documentation Corpus

`company_docs/bizpilot_synthetic_corpus.jsonl` is the active RAG dataset. It is a larger AI-synthesized JSONL corpus for a fictional but consistent BizPilot AI company. It was created after professor feedback so the RAG system can retrieve four or five related chunks for multi-part questions instead of answering from one short record. Each row contains metadata and cleaned content:

- document ID
- company
- document type
- title
- source URL
- retrieved date
- corpus type
- content

The first synthetic Markdown documents were moved to `company_docs_synthetic_archive/`. They are kept only as an archive of the first pipeline test and are not used by the active RAG pipeline.

The smaller public reference corpus is kept at:

```text
company_docs/public_saas_docs.jsonl
```

The active synthetic corpus can be regenerated with:

```powershell
.venv\Scripts\python scripts\generate_synthetic_company_corpus.py
```

### Lead Scoring

`lead_scoring/` contains the Kaggle lead-scoring dataset plan. Raw and processed datasets should be stored locally in:

- `data/lead_scoring/raw/`
- `data/lead_scoring/processed/`

These folders are ignored by Git to avoid committing downloaded datasets unless the dataset license clearly allows redistribution.

### CRM Sample Leads

`crm_sample_leads/crm_leads_sample.csv` is a small CRM-style sample dataset for the Week 3 integrated lead-scoring module. It contains lead identifiers, company details, lead source/origin fields, website engagement signals, contact permissions, and activity fields.

`crm_sample_leads/scored_crm_leads.csv` is generated by:

```powershell
.venv\Scripts\python src\lead_scoring_batch.py
```

## Türkçe

Bu klasör BizPilot AI projesi için kullanılan veri dosyalarını içerir.

### Company Documentation Corpus

`company_docs/bizpilot_synthetic_corpus.jsonl` aktif RAG datasetidir. Bu JSONL corpus, fictional ama tutarlı BizPilot AI şirketi için AI ile sentezlenmiş daha büyük bir corpus'tur. Profesör geri bildiriminden sonra eklendi; böylece RAG sistemi multi-part sorular için tek kısa kayıttan cevap vermek yerine dört veya beş ilgili chunk retrieve edebilir. Her satır metadata ve cleaned content içerir:

- document ID
- company
- document type
- title
- source URL
- retrieved date
- corpus type
- content

İlk sentetik Markdown dokümanları `company_docs_synthetic_archive/` klasörüne taşındı. Bu dosyalar sadece ilk pipeline testinin arşivi olarak tutulur ve aktif RAG pipeline tarafından kullanılmaz.

Daha küçük public reference corpus şu dosyada tutulur:

```text
company_docs/public_saas_docs.jsonl
```

Aktif synthetic corpus şu komutla yeniden üretilebilir:

```powershell
.venv\Scripts\python scripts\generate_synthetic_company_corpus.py
```

### Lead Scoring

`lead_scoring/` klasörü Kaggle lead-scoring dataset planını içerir. Raw ve processed dataset dosyaları lokal olarak şu klasörlerde tutulmalıdır:

- `data/lead_scoring/raw/`
- `data/lead_scoring/processed/`

Bu klasörler Git tarafından ignore edilir. Böylece dataset lisansı açıkça izin vermedikçe indirilen dataset dosyaları repository'ye eklenmez.

### CRM Sample Leads

`crm_sample_leads/crm_leads_sample.csv`, 3. hafta entegre lead-scoring modülü için küçük bir CRM-style sample datasetidir. Lead identifier, company details, lead source/origin alanları, website engagement sinyalleri, contact permissions ve activity alanları içerir.

`crm_sample_leads/scored_crm_leads.csv` şu komutla üretilir:

```powershell
.venv\Scripts\python src\lead_scoring_batch.py
```
